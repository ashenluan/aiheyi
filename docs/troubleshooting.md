## 1. LLM 文本模型相关

### 1.1 `API 返回 429`
| 项目 | 说明 |
|------|------|
| **含义** | Too Many Requests — API 服务商限流 |
| **常见原因** | 短时间内请求过多（如刚做完 AI 一键提取后立即翻译） |
| **解决** | ① 等 1-2 分钟后重试 ② 系统已内置 429 自动重试（3s/6s/12s，共 3 次） ③ 如频繁触发，考虑升级 API 套餐或换用限流更宽松的模型 |

### 1.2 `API 返回 401` / `API 返回 403`
| 项目 | 说明 |
|------|------|
| **含义** | API Key 无效或已过期 |
| **解决** | 进入「设置」页 → 检查 LLM API Key 是否正确、是否过期、是否有余额 |

### 1.3 `API 返回 400`
| 项目 | 说明 |
|------|------|
| **含义** | 请求格式错误 |
| **常见原因** | ① 模型名称填错（如把 `gemini-2.5-pro` 写成 `geminipro`） ② API URL 格式不对 |
| **解决** | 检查「设置」页的模型名称和 API URL 是否与服务商文档一致 |

### 1.4 `API 返回 500` / `502` / `503` / `504`
| 项目 | 说明 |
|------|------|
| **含义** | API 服务端/网关错误 |
| **常见原因** | ① API 代理 CDN 暂时不可用 ② 服务商维护中 |
| **解决** | ① 等几分钟后重试（系统已内置 500-504 自动重试 3 次） ② 切换到其他 API 预设（如国内通义千问） |

### 1.5 `LLM API 网关错误 ... CDN暂时不可用`
| 项目 | 说明 |
|------|------|
| **含义** | API 代理返回了 HTML 错误页面而非 JSON |
| **解决** | 切换到国内直连 API 预设（设置页 → LLM 预设 → 选择"通义千问"等），无需翻墙 |

### 1.6 `缺少必要参数: apiKey, model, prompt`
| 项目 | 说明 |
|------|------|
| **含义** | 设置页配置不完整 |
| **解决** | 进入「设置」页，确保 API Key、模型名称都已填写 |

### 1.7 `请先在设置页配置 LLM API Key`
| 项目 | 说明 |
|------|------|
| **含义** | 设置页 LLM API Key 为空 |
| **解决** | 进入「设置」→ 填写 API Key |

## 2. AI 一键提取相关

### 2.1 `没有可用的剧本内容`
| 项目 | 说明 |
|------|------|
| **含义** | 未找到足够长的剧本文本（<50字） |
| **解决** | ① 先在「剧本管理」页添加剧本 ② 如已添加，检查是否选择了正确的章节 ③ 检查 toast 中的"来源"信息定位问题 |

### 2.2 `AI提取超时（5分钟）`
| 项目 | 说明 |
|------|------|
| **含义** | 5 分钟内未收到 AI 响应 |
| **常见原因** | ① 剧本太长，AI 处理时间长 ② 网络不稳定 ③ API 服务商响应慢 |
| **解决** | ① 缩短剧本长度或选择单章节 ② 换用更快的模型（如 Flash 系列） ③ 检查网络连接 |

### 2.3 提取结果不完整（角色有、场景/道具为空）
| 项目 | 说明 |
|------|------|
| **含义** | AI 输出被截断，只提取了部分类别 |
| **系统行为** | `/api/extract` 已内置空类别自动补全重试（非流式 temperature=0.2） |
| **解决** | ① 通常系统会自动补全 ② 如仍为空，手动再点一次「AI 一键提取」③ 空类别不会覆盖已有数据 |

### 2.4 提取后刷新页面数据丢失
| 项目 | 说明 |
|------|------|
| **含义** | 提取结果未持久化到 IndexedDB |
| **系统行为** | 已修复 — 提取结果通过 `persistExtractResult()` 直接写入 KV |
| **解决** | 升级到最新版本 |

## 3. AI 翻译按钮相关

### 3.1 `请先填写中文外观描述`
| 项目 | 说明 |
|------|------|
| **含义** | 角色/场景/道具的「中文描述」为空 |
| **解决** | 先在描述文本框中输入中文外观描述，再点翻译 |

### 3.2 `AI 翻译超时（60s）`
| 项目 | 说明 |
|------|------|
| **含义** | 60 秒内（含 429 重试时间）未完成翻译 |
| **解决** | ① 等几分钟后重试 ② 换用更快的模型 |

### 3.3 `AI 翻译失败: API 返回 429: ...`
| 项目 | 说明 |
|------|------|
| **含义** | 3 次 429 自动重试后仍然限流 |
| **解决** | 等 2-3 分钟后重试，或升级 API 套餐 |

### 3.4 `AI 返回空内容`
| 项目 | 说明 |
|------|------|
| **含义** | API 请求成功但返回的文本为空 |
| **常见原因** | ① 模型安全过滤拦截了内容 ② 模型不支持该任务 |
| **解决** | ① F12 Console 查看 `[AI翻译] API 返回:` 日志 ② 换一个模型试试 |

### 3.5 翻译后英文区域没显示
| 项目 | 说明 |
|------|------|
| **含义** | 翻译成功但 `item.prompt` 渲染条件 `!== ""` 未满足 |
| **解决** | 检查 toast 是否显示「AI 翻译完成 ✓」；如是，刷新页面查看 |

## 4. Gemini Tab 生图相关

### 4.1 `无法连接 Gemini Tab 服务`
| 项目 | 说明 |
|------|------|
| **含义** | GeminiTab 进程未启动或端口 3099 被占用 |
| **解决** | ① 检查「Gemini Tab」页面是否已点击启动 ② `netstat -ano | findstr :3099` 查看端口占用 ③ `taskkill /f /im node.exe` 清理残留进程后重启 |

### 4.2 `同步失败: Unknown action`
| 项目 | 说明 |
|------|------|
| **含义** | GeminiTab 后端版本过旧，不支持该 action |
| **常见原因** | 旧版 GeminiTab 进程残留（如手动运行了 `启动GeminiTab.bat`） |
| **解决** | ① 关闭所有旧 GeminiTab 窗口 ② `taskkill /f /im node.exe` ③ 重启飞彩工作室 |

### 4.3 参考图上传失败 / 图片没有上传到 Gemini
| 项目 | 说明 |
|------|------|
| **常见原因** | ① Chromium 开启了翻译功能 ② Gemini 页面语言变体不在匹配列表中 |
| **解决** | ① 在 Chromium 中设置「一律不翻译英文」 ② 检查 `debug-screenshots/` 下的截图 |

### 4.4 图片提取为缩略图（模糊/小图）
| 项目 | 说明 |
|------|------|
| **含义** | Google CDN URL 中的尺寸参数导致返回缩略图 |
| **系统行为** | 已修复 — `upgradeGoogleCdnUrl()` 自动替换为 `=s0` 请求原图 |
| **解决** | 升级到 V1.15+ 版本 |

### 4.5 `AbortSignal.any is not a function`
| 项目 | 说明 |
|------|------|
| **含义** | EXE 内置 Node.js 版本 < 20.3.0 |
| **系统行为** | V1.81+ 已全部替换为手动 polyfill |
| **解决** | 升级到 V1.81+ 版本 |

| 项目 | 说明 |

| **含义** | Playwright Headless Shell 未安装 |
| **系统行为** | V1.161+ 已内置 Headless Shell |

### 5.2 `Target page, context or browser has been closed`
| 项目 | 说明 |
|------|------|
| **含义** | Chromium 浏览器崩溃 |
| **历史原因** | `--single-process` 参数导致（已在第八次迭代移除） |
| **解决** | 升级到最新版本；如仍出现，检查系统内存是否不足 |

### 5.3 `未获取到记录ID`

|------|------|
| **含义** | 即梦 API 返回成功但无 aigcData |

| **解决** | ① 重新登录即梦网站 ② 检查 `outputs/seedance-log.txt` 查看详细返回 ③ 修改提示词避免违禁词 |

### 5.4 `你输入的文字不符合平台规则`

| **含义** | 即梦内容安全过滤器拦截 |

### 5.5 `face_check_error (ret=4010)`

### 5.6 `积分不足 (ret=5000)`
| 项目 | 说明 |
|------|------|
| **含义** | 即梦账户积分余额不足 |

## 10. Mac 版相关

### 10.1 `bad interpreter: /bin/bash^M`
| 项目 | 说明 |
|------|------|

### 10.4 Mac Node.js 未安装
| 项目 | 说明 |
|------|------|
| **解决** | 双击 `install.command`，会自动通过 Homebrew 安装 Node.js |

## 通用排查步骤

### 如何查看详细日志

| 位置 | 查看方法 | 包含的信息 |
|------|---------|-----------|
| **浏览器 Console** | F12 → Console 面板 | `[AI翻译]` `[AI提取]` `[auto-save]` 前端日志 |
| **Dev Server 终端** | 启动 EXE 的命令行窗口 | `[LLM API]` `[Pipeline LLM]` `[browser]` 后端日志 |
| **Seedance 日志文件** | `outputs/seedance-log.txt` | 即梦 API 请求/响应详情 |

### 一键清理残留进程（Windows）
```powershell
taskkill /f /im node.exe
# 然后重新启动飞彩工作室
```

### 一键清理残留进程（Mac）
```bash
pkill -f node
# 然后双击 启动飞彩工作室.command
```

### 检查设置页配置

| 配置项 | 位置 | 必填 |
|--------|------|------|
| LLM API Key | 设置 → LLM 配置 | ✅ |
| LLM API URL | 设置 → LLM 配置 | ✅ |
| LLM 模型名称 | 设置 → LLM 配置 | ✅ |
| 即梦 Cookie | 自动获取（浏览器登录后） | Seedance 必需 |

## 错误码速查表

| HTTP 状态码 | 含义 | 处理 |
|------------|------|------|
| 400 | 请求参数错误 | 检查设置页配置 |
| 401 | API Key 无效 | 更换 API Key |
| 403 | 无权限 | 检查 API Key 权限 |
| 429 | 请求限流 | 等 1-2 分钟后重试 |
| 500 | 服务端错误 | 重试或换模型 |
| 502 | 网关错误 | 换 API 预设 |
| 503 | 服务不可用 | 稍后重试 |
| 504 | 网关超时 | 缩短输入或换模型 |

| 即梦错误码 | 含义 | 处理 |
|-----------|------|------|
| ret=1000 | 请求被拒绝 | 升级版本 |
| ret=4010 | 人脸检测 | 完成人脸授权 |
| ret=5000 | 积分不足 | 充值积分 |
